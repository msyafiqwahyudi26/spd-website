## 2. Audit Backend API

### ✅ Lengkap dan Solid

**25 API Routes tersedia:**
```
/api/auth           — login, refresh, logout, me
/api/publications   — CRUD + view counter + PDF
/api/events         — CRUD + slug + startsAt
/api/programs       — CRUD + status/category/fullContent/gallery
/api/contacts       — submit + list + update status
/api/users          — CRUD + role management
/api/analytics      — page views, stats
/api/logs           — audit trail semua aksi admin
/api/settings       — site-wide config (single row)
/api/categories     — kategori publikasi
/api/media          — upload + library + delete
/api/partners       — CRUD + logo
/api/stats          — statistik homepage
/api/team           — CRUD + featured flag
/api/milestones     — perjalanan organisasi
/api/missions       — butir-butir misi
/api/approaches     — pendekatan/feature cards
/api/core-values    — nilai-nilai organisasi
/api/footer-links   — tautan footer per section
/api/system         — health check + email test
/api/annual-reports — laporan tahunan + PDF
/api/subscribers    — newsletter subscriber list
/api/infografis     — gambar infografis pemilu
/api/kpu            — proxy KPU Sirekap (dengan cache)
```

**Keamanan Backend:**
- ✅ JWT auth via httpOnly cookie (tidak di localStorage)
- ✅ Helmet security headers
- ✅ CORS dibatasi per origin
- ✅ Rate limiting (sliding window, per-IP)
- ✅ Input validation (`lib/validate.js`)
- ✅ Role-based access (admin vs publisher)
- ✅ Audit log setiap aksi create/update/delete
- ✅ Multer upload dengan whitelist tipe file

**Database Schema (SQLite + Prisma):**
```
User, Publication, Program, Event, Contact, Log,
Setting, Category, Media, Partner, Stat, TeamMember,
Milestone, CoreValue, Approach, FooterLink, MissionItem,
AnnualReport, Subscriber, Infografis
```
Total: 20 model — sangat komprehensif.

### ⚠️ Masalah dan Risiko

**[KRITIS] Email tidak dikonfigurasi**
```
EMAIL_HOST, EMAIL_PORT, EMAIL_USER, EMAIL_PASS, EMAIL_FROM
```
- Variabel env ini tidak ada di `.env` server
- Akibat: Pesan kontak masuk TIDAK dikirim notifikasi email ke admin
- Subscriber baru TIDAK menerima konfirmasi
- **Solusi:** Konfigurasi SMTP (Gmail App Password / Mailgun / Resend)

**[SEDANG] SQLite di produksi**
- SQLite aman untuk traffic rendah-menengah, tapi:
  - Tidak support concurrent writes yang tinggi
  - Backup manual (file `prod.db`)
  - Tidak bisa scale horizontal
- **Saran:** Pertimbangkan migrasi ke PostgreSQL jika traffic meningkat

**[SEDANG] Rate limiting in-memory**
- Rate limiter menggunakan `Map()` di memori
- Reset ketika `pm2 restart` dilakukan
- Tidak efektif jika ada multiple Node processes
- **Saran:** Ganti dengan Redis-backed store untuk produksi yang lebih ketat

**[RENDAH] KPU Sirekap API offline**
- `sirekap-obj-data.kpu.go.id` tidak dapat diakses dari server
- Graceful degradation sudah berfungsi (amber banner)
- Data tetap menampilkan statistik statis
- Perlu monitoring apakah KPU API kembali online

**[RENDAH] JWT secret hardcoded fallback**
- Jika `JWT_SECRET` tidak di-set, ada fallback value
- Pastikan `JWT_SECRET` selalu di-set dengan nilai kuat di `.env`

## 5. Audit Keamanan

### ✅ Sudah Diimplementasikan
- httpOnly JWT cookie (tidak di localStorage)
- CORS origin whitelist
- Helmet headers (XSS, CSRF protection)
- Input sanitization di validator
- Role-based route guard (`RequireAdmin`)
- Rate limiting per-IP pada semua API
- File upload whitelist (hanya image/PDF)
- Password bcrypt hash

### ⚠️ Risiko yang Perlu Diperbaiki

**[TINGGI] `.env` tidak ada di git tapi perlu backup**
- Jika server hilang, kredensial JWT_SECRET hilang
- Semua session akan invalid setelah deploy baru
- **Saran:** Backup `.env` di password manager atau vault

**[SEDANG] Tidak ada HTTPS enforcement di backend**
- Backend diakses via Nginx sebagai reverse proxy
- Verifikasi bahwa Nginx redirect HTTP → HTTPS

**[SEDANG] Session management**
- Token di httpOnly cookie tapi tidak ada refresh token mechanism
- Jika token expired, user langsung di-logout
- **Saran:** Implementasi refresh token untuk UX yang lebih baik

**[RENDAH] Tidak ada 2FA/MFA**
- Login hanya dengan email + password
- **Saran:** Tambahkan TOTP (Google Authenticator) untuk akun admin

## 8. Teknologi Stack

  Tailwind CSS v4
  i18n (ID/EN) via Context
  No external UI library (custom components)

Backend:
  Node.js 20 + Express
  Prisma ORM + SQLite (prod.db)
  JWT (httpOnly cookie)
  Multer (file upload)
  Nodemailer (email — belum dikonfigurasi)

  Ubuntu 24.04 LTS
  Nginx reverse proxy (port 80/443 → 3001)
  PM2 process manager
  GitHub Actions / manual deploy via git pull
